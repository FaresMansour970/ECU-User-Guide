00 — مقدمة

نظرة عامة على البرنامج

ECU Intelligence Pro v5 هو برنامج متخصص لتحليل ومعايرة ملفات BIN الخاصة بوحدات التحكم الإلكترونية (ECU) في السيارات. البرنامج مبني على معمارية 7 مراحل متسلسلة تضمن الدقة والأمان في كل عملية.

تحليل ملفات BIN

تحسين أداء المحرك بثلاث مراحل مع حدود أمان ذكية
تعلم الفروقات

تنفيذ تعديلات مخصصة بالنمط مع التحقق السياقي المتقدم
إدارة الأعمال CRM
سجل عملاء كامل وتوثيق الأعمال مع تصدير CSV
متطلبات النظام: Windows 10/11 (64-bit) · Python 3.10+ · PyQt6 · 4GB RAM · 500MB
01 — التثبيت

كيفية تشغيل البرنامج

البرنامج لا يتطلب تثبيتاً تقليدياً. يعمل مباشرة من مجلده مع Python وPyQt6 مثبتَين على الجهاز.

1
ضع ملفات البرنامج في مجلد واحد
ضع جميع الملفات في مجلد مثل C:\ECU_Pro\. تأكد من صلاحيات الكتابة لأن البرنامج يحتاج لإنشاء قواعد بيانات.
2
تثبيت المتطلبات
افتح Command Prompt ونفّذ:
CMD / PowerShell 
pip install PyQt6 pyqt6-qt6 pyqt6-sip
3
تشغيل البرنامج
من نفس المجلد نفّذ:
CMD 
python ecu_intel_v5.py
ملاحظة: عند أول تشغيل ينشئ البرنامج تلقائياً: output/ وoutput/backups/ وscripts/ وقواعد البيانات .db. لا تحذف هذه الملفات.
نصيحة: أنشئ ملف run.bat يحتوي على python ecu_intel_v5.py للتشغيل بنقرة مزدوجة.

03 — خط الأنابيب

فهم خط الأنابيب السباعي

كل مرة تضغط "تحليل وضبط تلقائي" يمر ملف BIN عبر 7 مراحل متتالية. فهم هذه المراحل يساعدك على قراءة نتائج التحليل بشكل صحيح.

1
SW ID Extraction — استخراج معرّف البرنامج
الأولوية: تحديد هوية الملف قبل أي شيء

البرنامج يبحث في ملف BIN عن مراسٍ مثل SW: وVER: و0261 و0262. عند العثور على نص مناسب يُستخرج كمعرّف برنامج لمطابقة السكريبتات لاحقاً.

إذا لم يُكشف SW ID تلقائياً، اذهب لـ Script Engine ← "Scan SW IDs" لرؤية جميع المرشحين.
2
Feature Detection — كشف الميزات
أهم مرحلة: ما الذي يوجد فعلاً في الملف؟
الميزةالتوقيعاتالحد الأدنى
DPF01 00 01, DPF, FAP2 توقيعات+
EGR01 01 01, EGR, 45 47 522 توقيعات+
SCR/AdBlueSCR, AdBlue, UREA1 توقيع
IMMOA5 A5 A5 A5, IMMO1 توقيع
"not detected" هو تحذير وليس منعاً — تأكد من صحة الملف قبل المتابعة.
6

ضروري لقبول الملف من وحدة التحكم
الخوارزميةالوصفالاستخدام

7
Diff Learning — حفظ الفروقات وتوليد السكريبتات
البرنامج يتعلم من كل عملية

بعد كل عملية ناجحة يحفظ البرنامج الفروقات في diff_learning.db لتوليد سكريبتات JSON تلقائية.

كلما تكررت العملية على نفس المتغير، كلما زادت درجة الثقة وأصبح البرنامج يقترح السكريبت تلقائياً.

08 — خرائط الوقود

تبويب خرائط الوقود

يعرض خريطة الوقود كجدول 16×16 خلية مع تلوين بصري. يمكن تعديل الخلايا يدوياً وتصديرها CSV.

1
تحديد الإزاحة (Offset)
حقل Offset يحدد بداية قراءة الخريطة. يُملأ تلقائياً إذا كُشف المتغير. التنسيق: 0x8000.
2
ضبط الأبعاد ونوع البيانات
Rows/Cols (افتراضي 16×16). نوع البيانات: uint16_le لـ Bosch، uint8 للقديمة، float32_le لـ Continental.
3
استخراج الخريطة
انقر Extract Map. الأحمر = قيم عالية، الأزرق = منخفضة. انقر Before/After لرؤية ملخص التغييرات.
4
تصدير CSV
انقر Export CSV لحفظ الخريطة كملف Excel/CSV للتوثيق أو المقارنة.
09 — Boost Map

تبويب Boost Map

عرض وتعديل خريطة ضغط الشاحن (Turbo Boost) مباشرة. التعديل يؤثر على ضغط الشاحن في كل نقطة تشغيل.

تحذير: الزيادة فوق 2.8 bar قد تتلف الشاحن أو الميكانيكا. البرنامج يُنبّه تلقائياً عند تجاوز هذا الحد.
1
تحميل البيانات من BIN
الخريطة لا تُحمَّل تلقائياً

تأكد من الإزاحة الصحيحة ثم انقر Load from BIN. الأزرق = ضغط منخفض، الأحمر = عالٍ.

2
تعديل الخلايا يدوياً
انقر مزدوج على أي خلية
القيمة / 10000 = bar. مثلاً 18000 = 1.8 bar.
3
تطبيق التعديلات على BIN
احفظ في ملف جديد

انقر Apply to BIN. يُحفظ في output/boost_map_YYYYMMDD.bin.

هذا الزر لا يحدّث Checksum تلقائياً. استخدم Script Engine أو أداة خارجية.
10 — DPF / EGR / IMMO / Tune

DPF / EGR / IMMO / Auto-Tune

يجمع عمليات الحذف والضبط في مكان واحد. يمكن تطبيق عمليات متعددة دفعة واحدة.

DPF Delete
يكتب 00 00 00 في إزاحة DPF
يعطّل طلب التجديد التلقائي
يحذف كودات P2002/P2463
يتطلب إزالة DPF الفيزيائي
غير قانوني للطرق العامة في بعض البلدان
EGR Delete
يكتب 00 00 00 في إزاحة EGR
يغلق صمام EGR برمجياً
يمنع رسائل الخطأ EGR
يُنصح بإغلاق الصمام ميكانيكياً أيضاً
Stage 3 تحذير: تحذف حد العزم وتزيد الوقود 30%. تتطلب ترقية ميكانيكية (injectors, turbo, intercooler). لا تطبّق على محرك أصلي.

البرنامج لا يتعرف على متغير ECU — "No confident match"
استخدم Override Variant لاختيار المتغير يدوياً. تحقق من أن الملف سليم وغير مشفر. ابحث في Hex View عن اسم المتغير.
خطأ "Patch validation failed"
البايتات في الإزاحة لا تطابق حقل Find. أسباب: الملف معدَّل مسبقاً، أو الإزاحة خاطئة، أو إصدار مختلف. استخدم Hex View للتحقق أولاً.
كيف أعرف أن الملف المعدَّل سليم؟
قارن بـ Diff Viewer. افتحه في تبويب التحليل وتأكد من قراءة المتغير. Entropy يجب أن يكون قريباً من الأصلي (فرق أقل من 0.1). اختبر على الديناميك أولاً.
غيّرت جهازي — كيف أنقل الترخيص؟
أرسل HWID الجهاز الجديد لفارس منصور على +970595438486. ستستلم مفتاحاً جديداً مرتبطاً بالجهاز الجديد.
البرنامج لا يفتح — خطأ Python أو PyQt6
تحقق: Python 3.10+ (python --version). PyQt6 مثبّت (pip install PyQt6). شغّل CMD كـ Administrator. تأكد من صلاحيات الكتابة.
كيف أضيف متغير ECU جديد؟
تواصل مع فارس منصور مع الإزاحات الضرورية. يمكن أيضاً كتابة سكريبت JSON مخصص في Script Engine يعمل على أي متغير.

الدعم متاح 24/7. أرسل هذه المعلومات لتسريع الحل.

ما ترسله عند طلب الدعم

1. Machine ID (HWID) من تبويب الدعم
2. متغير ECU (مثلاً EDC17C10)
3. وصف المشكلة بالتفصيل
4. رسالة الخطأ كاملة من السجل
5. ملف BIN (اختياري لكن يسرّع كثيراً)
